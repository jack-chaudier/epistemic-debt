# coarseness — does the reason-channel gap track answer-quotient coarseness?

Design and predictions: [prereg_coarseness.md](prereg_coarseness.md). Two grok-only arms, single-condition (p=1) vs six-condition (p=6) policies, 15-word contract-blind compression, same probes. Tests the LLM-scale shadow of the Shelf Width Law (`M_k = Q_(k,1)`: at p=1 the verdict logically determines the reason).

Verdict: **P-S2 PASS, P-S1 and P-S3 FAIL — and the failure is the interesting result.** At p=6, witness loss crashes reason-naming to **0/16** (guessing), as predicted. At p=1, the clean theory says the single failing parameter is deducible from "DENIED + one-condition policy" without the value; the model does **not** do this — it names the reason on only 2/13 lost items and abstains on 10/13. Two facts explain and deepen it: (a) the p=1 verdict is a pure always-DENY prior (APPROVED accuracy 0/6), so "decision-lost = 1.000" carries zero evidential content; (b) the model treats *name-the-reason* as **retrieval of the value from context**, not **inference from verdict + policy structure**.

The honest upgrade to the theory: the idealized quotient identity `M_k = Q_(k,1)` does **not** transfer to LLM behavior. Even when the witness is logically recoverable from the answer (fiber size 1), the empirical reason channel stays collapsed — the dissociation is *more* severe than the fibration predicts, because these models don't perform the logical collapse. For context engineering: a compressed note that logically pins down its own cause will still not yield that cause on questioning unless the deciding value is explicitly present. Recorded as a REFUTED clean prediction with a sharper replacement claim (theory doc Appendix G).
